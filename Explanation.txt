📌 What Is Happening in Your Project (Point-wise)

You trained three different CNN models:

V1 – Advanced CNN

V2 – CNN with Data Augmentation

V3 – Transfer Learning (MobileNetV2)

Each model learns internal visual representations of images, not just final class labels.

These internal representations are called deep features.

Deep features are numerical vectors that describe:

Edges

Textures

Shapes

Object-level patterns

📌 Why Feature Extraction Was Done

Final model predictions (Cat/Dog) are not sufficient for scientific comparison.

To understand how well each model learns, we extract features before the final classification layer.

This allows comparison of:

Representation quality

Feature richness

Generalization ability

📌 What Happens During Feature Extraction

A trained model is loaded (V1, V2, or V3).

The last classification layer is removed.

The remaining network outputs a feature vector for each image.

Each validation image is passed through the network.

The output is a fixed-length numeric vector per image.

📌 What Is Stored in Each CSV File

Each row represents one image.

Columns represent learned deep features.

An additional column stores the true label (Cat/Dog).

Another column stores the model version (V1 / V2 / V3).

📌 Why Separate CSV Files Were Created

Each model learns different types of features.

Separate CSV files preserve model-specific representations.

This enables fair and isolated analysis per architecture.

📌 Why CSV Files Were Combined

All model feature datasets were merged into one file.

This creates a unified comparison dataset.

It allows:

Cross-model analysis

Feature distribution comparison

Visualization (PCA / t-SNE)

📌 Advantages of This Approach

Moves the project beyond simple accuracy comparison.

Enables data-driven evaluation of learning quality.

Supports research-level justification of results.

Allows reuse of features for classical ML algorithms.

📌 How This Helps in Viva / Review

Demonstrates understanding of CNN internals.

Shows systematic experimentation across versions.

Proves architectural improvements quantitatively.

Positions the project closer to real research work.
